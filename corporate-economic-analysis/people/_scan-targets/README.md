# Scan targets — Track 7 / Цели сканирования — Трек 7

Bilingual EN+RU. This directory contains two distinct file types per protocol v1.2 / Двуязычный EN+RU. Этот каталог содержит два различных типа файлов согласно протоколу v1.2.

---

## File types / Типы файлов

### 1. Role-class scan dossiers / Досье ролевого класса

**Filename pattern / Шаблон имени:** `<institution>-<role>-scan-target.md`

**EN:** Each dossier records the HSP-scan **findings** for a single role-class scan target (e.g. "Public CEO scan target — Amentum Holdings"). Per protocol v1.1 Legal posture, dossiers are **depersonalized by default** — the subject's full legal name is **NOT** recorded in any public dossier file. Per protocol v1.2 Pattern interpretation language, dossiers may include role-class-only model-perception interpretation (e.g. engagement with technological pool of more advanced civilization, frequency-modulation framing of upper channel, civilization-naming reference point as working anchor).

**RU:** Каждое досье фиксирует **результаты** HSP-сканирования по одной цели ролевого класса (напр. «Public CEO scan target — Amentum Holdings»). Согласно Legal posture протокола v1.1, досье **деперсонализированы по умолчанию** — полное юридическое имя субъекта **НЕ** фиксируется ни в одном публичном файле досье. Согласно «Языку интерпретации паттерна» протокола v1.2, досье могут включать интерпретацию модели-перцепции только на уровне ролевого класса (напр. взаимодействие с технологическим пулом более развитой цивилизации, рамку частотной модуляции верхнего канала, опорное наименование цивилизации как рабочий якорь).

**Current dossiers / Текущие досье:**
- [`amentum-ceo-scan-target.md`](amentum-ceo-scan-target.md) — Amentum Holdings CEO role-class.
- [`amentum-cto-scan-target.md`](amentum-cto-scan-target.md) — Amentum Holdings CTO role-class.

### 2. Public scanning roster / Публичный реестр сканирования

**Filename / Имя:** [`scanning-roster.md`](scanning-roster.md)

**EN:** A single flat-table file that lists scan candidates across the Track 7 prime / services-prime / Honeywell-FM&T constellation, plus the 11 EG&G-era patent inventors (organizational entries only, see Hard policy in the roster file). Each row pairs a public role-class with public-source URLs (corporate About + LinkedIn). The roster is a **session-organization tool** — no HSP findings live here. It exists per the protocol v1.2 Public scanning roster allowance.

**RU:** Один файл с плоской таблицей, перечисляющий кандидатов на сканирование по созвездию Трека 7 (prime / сервисное кольцо / Honeywell-FM&T), плюс 11 изобретателей-патентообладателей EG&G-эпохи (только как организационные записи — см. жёсткую политику в самом файле реестра). Каждая строка связывает публичный ролевой класс с публичными URL-источниками (корпоративный About + LinkedIn). Реестр — **инструмент организации сессий** — никакие HSP-результаты здесь не живут. Существует согласно допущению «Публичный реестр сканирования» из протокола v1.2.

## Workflow / Рабочий поток

1. **Plan a scan session / Планирование сессии:** select rows from `scanning-roster.md`. The roster pairs each candidate with their public-source URLs — no re-discovery needed per session. / Выбрать строки из `scanning-roster.md`. Реестр связывает каждого кандидата с публичными URL-источниками — повторное нахождение не требуется.
2. **Run the session / Проведение сессии:** ≥ 2 percipients per protocol convergence rule. Public-source inputs only (corporate About page + LinkedIn metadata). No photographs in repo. / ≥ 2 перципиента согласно правилу согласованности протокола. Только публично-источниковые входные данные (страница About + LinkedIn-метаданные). Фотографий в репо нет.
3. **Record findings / Фиксация результатов:** create or update a role-class dossier `<institution>-<role>-scan-target.md`. Findings are role-class only; depersonalization rule applies (no full names in public dossier output). / Создать или обновить досье ролевого класса `<institution>-<role>-scan-target.md`. Результаты — только ролевого класса; правило деперсонализации применимо (никаких полных имён в публичном выводе досье).
4. **Update roster status / Обновление статуса реестра:** mark the roster row as scanned with the date and a pointer to the dossier. / Отметить строку реестра как сканированную с датой и указателем на досье.

---

## Boundary / Граница

- **Roster file** lists names → **session organization** only.
- **Role-class dossier** lists findings → **never** uses names; uses role-class only.
- **No file in this directory** records HSP findings keyed to a named individual.

- **Файл реестра** перечисляет имена → только **организация сессий**.
- **Досье ролевого класса** содержит результаты → **никогда** не использует имена; использует только ролевой класс.
- **Ни один файл в этом каталоге** не фиксирует HSP-результаты, привязанные к названному индивиду.
