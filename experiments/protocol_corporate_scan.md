## Purpose / Назначение

The HSP Corporate Scan Protocol establishes standardized procedures for collecting and documenting high-sensitivity-perception (HSP) impressions from qualified percipients regarding **public-figure C-level executives** of publicly-listed institutions. The protocol is a **research instrument**, not a diagnostic tool. It investigates hypothesised correlations between corporate roles in the classified-aerospace-contractor space (Track 7 graph) and HSP-percipient impressions of caste, chakra activation, upper-channel target, and implant / boost / shielding markers. **Subject scope:** publicly-named C-level officers of publicly-listed institutions — never private individuals.

Протокол HSP-сканирования корпоративных руководителей устанавливает стандартизированные процедуры сбора и документирования впечатлений высокочувствительного восприятия (HSP) от квалифицированных перципиентов в отношении **публичных C-level руководителей** публично-листингованных институций. Протокол является **исследовательским инструментом**, а не диагностическим средством. Он исследует гипотетические корреляции между корпоративными ролями в пространстве засекреченных аэрокосмических подрядчиков (граф Трека 7) и HSP-впечатлениями перципиентов о касте, активации чакр, верхне-канальной цели и маркерах имплантов / усилений / экранирования. **Область субъектов:** только публично-названные C-level руководители публично-листингованных институций — никогда частные лица.

## Subject inclusion criteria / Критерии включения субъекта

A subject is **eligible** only if ALL of the following are true / Субъект **допустим** только при выполнении ВСЕХ условий:

1. The subject is a **publicly-named officer** of a **publicly-listed company** (or major public-sector institution with a public officer registry). / Субъект — **публично названный руководитель** **публично-листингованной компании** (или крупного публичного института с открытым реестром).
2. At least **one public photograph** is available from a public source (LinkedIn / corporate "About" / press conference / official press release). / Доступна как минимум **одна публичная фотография** из открытого источника (LinkedIn / корпоративный «About» / пресс-конференция / пресс-релиз).
3. The institution is **relevant to Track 7** corporate-economic analysis. / Институция **релевантна Треку 7** — корпоративно-экономическому анализу.
4. The subject's **full name and current corporate role** are public. / **ФИО и текущая корпоративная роль** субъекта публичны.

**Hard exclusions / Жёсткие исключения:** NO private individuals / нет частным лицам; NO unlisted entities / нет непубличным организациям; NO subject without ≥1 public photograph in the input dossier / нет субъектам без ≥1 публичной фотографии во входном досье; NO content based on private personal data of any kind / нет материалам на основе личных приватных данных любого рода.

## Subject dossier inputs / Входные данные досье

**Default mode / Режим по умолчанию:** subjects are referenced by **role-class only** ("CEO of <employer>", "CTO of <employer>") in public dossiers. **Full name may be retained in private working notes** but **MUST NOT appear in any public-facing scan dossier file**. / Субъекты обозначаются **только по ролевому классу** («CEO компании <Х>», «CTO компании <Х>») в публичных досье. **Полное имя может сохраняться в приватных рабочих заметках**, но **НЕ ДОЛЖНО появляться ни в одном публичном файле досье сканирования**.

| Field / Поле | Source / Источник |
|--------------|-------------------|
| Public photograph (URL reference only) / Публичная фотография (только URL-ссылка) | LinkedIn / corporate "About" / press conference; **NOT stored in repo** / в репо **не сохраняется** |
| LinkedIn URL | Public profile only — link retained in private working notes; **NOT** copied into public dossier / только публичный профиль — ссылка сохраняется в приватных рабочих заметках; в публичное досье **НЕ** копируется |
| Full name / ФИО | **Optional — NOT recorded in public dossier output** / **Опционально — в публичном выводе досье НЕ фиксируется** |
| Current corporate role / Текущая роль | Corporate "About" / SEC filings — role-class is the public key |
| Employer-of-record / Работодатель | Corporate registry / корпоративный реестр |

**NO private personal data of any kind** is permitted in the input dossier. **NO photographs are stored in the repository** — only references to public sources. / **Никакие личные приватные данные** во входном досье не допускаются. **Фотографии в репозитории не сохраняются** — только ссылки на публичные источники.

## Convergence analysis / Сходимость

- **Minimum 2 independent percipients per subject. / Минимум 2 независимых перципиента на субъекта.**
- For the **"convergent"** tag: agreement on **caste** AND **chakra-activation profile** is required. / Для тега **«convergent»** требуется согласие по **касте** И **профилю активации чакр**.
- **Divergent observations** are reported transparently in a separate sub-section, never suppressed. / **Расхождения** прозрачно фиксируются в отдельном подразделе и никогда не замалчиваются.

---

## Falsifiability / Фальсифицируемость

HSP scan results are **explicitly NOT diagnostic** and **cannot be empirically verified at present**. Outputs are stored as **percipient-observation logs**, not as claims about the subject. The subject does **not** consent to and does **not** know about the scan; therefore **no actionable claim** is made about the subject as an individual. All input data is fully public; no private data is solicited or used. / Результаты HSP-сканирования **явным образом НЕ диагностичны** и **не могут быть эмпирически верифицированы на текущий момент**. Вывод хранится как **журнал наблюдений перципиента**, а не как утверждение о субъекте. Субъект **не** даёт согласие и **не** знает о сканировании; поэтому **никакое action-able-утверждение** о субъекте как о личности не делается. Все входные данные полностью публичны; приватные данные не запрашиваются и не используются.

## Ethical posture / Этическая позиция

- Subjects are **public figures only**. **No contact**. **No interference**. **No claim** about personal life. / Субъекты — **только публичные фигуры**. **Никакого контакта**. **Никакого вмешательства**. **Никаких утверждений** о личной жизни.
- Results are framed at the **institutional layer** (Track 7 graph), not at the personal layer. / Результаты оформляются на **институциональном уровне** (граф Трека 7), а не на личностном.

## Legal posture / Юридическая позиция

The protocol is written to **eliminate defamation, DMCA, and reputational-harm risk** at the public-output layer. The rules below are absolute and override any percipient impression to the contrary. / Протокол написан так, чтобы **исключить риски диффамации, DMCA и репутационного ущерба** на уровне публичного вывода. Правила ниже абсолютны и перекрывают любое впечатление перципиента, противоречащее им.

### Hard rule — depersonalization-by-default / Жёсткое правило — деперсонализация по умолчанию

**EN:** Public dossiers reference subjects by **role-class only**. The dossier filename is `<institution>-<role>-scan-target.md`; the title and subject card use role-class wording ("Public CEO scan target — <institution>"). The subject's full legal name is **NOT** present in any public-facing file in the repository. Personal name may be retained in private working notes for internal correlation; it does NOT cross the public boundary.

**RU:** В публичных досье субъект обозначается **только ролевым классом**. Имя файла досье — `<institution>-<role>-scan-target.md`; заголовок и карточка субъекта используют ролевую формулировку («Public CEO scan target — <institution>»). Полное юридическое имя субъекта **НЕ** присутствует ни в одном публичном файле репозитория. Имя может сохраняться в приватных рабочих заметках для внутренней корреляции; через публичную границу оно не переходит.

### Required language / Обязательный язык

- **Use:** "model perception" / "symbolic interpretation" / "role-class signal" / "перцепция модели" / «символическая интерпретация» / «сигнал ролевого класса».
- **Do NOT use:** "scanning of a person" / "сканирование человека" / "scanned the CEO" / "просканировали CEO".
- **Use:** "the CEO scan target" / "the role-class signal" / «цель сканирования CEO» / «сигнал ролевого класса».
- **Do NOT use:** any phrasing that names a specific individual + makes a content claim about them.

### Hard ban list / Жёсткий стоп-лист

The following claims are **NEVER** recorded in any public dossier, regardless of disclaimer count, percipient confidence, or convergence:

1. Claims (or hints, or "percipient impressions") about a named individual's **psychology, intentions, or hidden motives**. / Утверждения (или намёки, или «впечатления перципиента») о **психологии, намерениях или скрытых мотивах** названного индивида.
2. Claims that a named individual **participates in any classified program** (UAP-related or otherwise) **as fact**. / Утверждения, что названный индивид **участвует в какой-либо засекреченной программе** (связанной с UAP или иной) **как факт**.
3. Claims that a named individual is **connected to** UAP / non-human technology / reverse-engineering programs as fact. / Утверждения о **связи** названного индивида с программами UAP / нечеловеческой технологии / обратной разработки как факт.
4. Claims about **internal properties, hidden activities, or undisclosed affiliations** of a named individual. / Утверждения о **внутренних свойствах, скрытой деятельности или нераскрытых аффилиациях** названного индивида.
5. **Verbatim percipient impressions** that, even when wrapped in disclaimers, can be read as defamation-class statements about a named individual. Such impressions, if they arise during a scan, are **NOT** transcribed into the public dossier. They may be retained in private working notes; the public dossier records only the abstracted role-class register. / **Дословные перципиентные впечатления**, которые даже в обёртке дисклеймеров могут читаться как утверждения класса диффамации о названном индивиде. Такие впечатления, если возникают в ходе сканирования, в публичное досье **НЕ** транскрибируются. Они могут сохраняться в приватных рабочих заметках; в публичное досье попадает только абстрагированный регистр ролевого класса.

### Photographs in repository / Фотографии в репозитории

**NO photographs of subjects** are stored in the repository under any circumstances. References point only to the **public source** (corporate "About" page URL / public LinkedIn profile URL). Authorial copyright + ToS of LinkedIn dictate this rule. / **Никакие фотографии** субъектов в репозитории не сохраняются ни при каких обстоятельствах. Ссылки указывают только на **публичный источник** (URL корпоративной страницы «About» / URL публичного LinkedIn-профиля). Авторское право + ToS LinkedIn диктуют это правило.

### Why these rules / Почему эти правила

These rules were established 2026-04-29 in response to a structured legal-risks assessment. The assessment is preserved in `corporate-economic-analysis/raw/banchenko_2026-04-26_legal_risks_assessment.md`. The conclusion is unambiguous: **even with multiple layers of disclaimer, naming a specific person + making content findings about them in a UAP-domain context creates defamation-class and DMCA risk that disclaimers do not neutralize.** The protocol's response is to remove the named-individual layer from public outputs entirely. / Правила введены 29.04.2026 в ответ на структурную оценку юридических рисков. Оценка сохранена в `corporate-economic-analysis/raw/banchenko_2026-04-26_legal_risks_assessment.md`. Вывод однозначен: **даже при многослойных дисклеймерах, указание конкретного человека + содержательные находки о нём в UAP-контексте создают риск класса диффамации и DMCA, который дисклеймеры не нейтрализуют.** Ответ протокола — удалить слой названного индивида из публичных выводов полностью.
